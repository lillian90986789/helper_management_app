# 本周菜单展示与提前排菜 — Design

## 背景

雇主目前只能把菜谱"安排到今日菜单"（`RecipeDetail.jsx` → `api.recipeToMeal`），`EmployerHome` / `MaidToday` 首页卡片也只展示"今天"这一天的 `MealOrder`。雇主没法提前几天把一周的菜排好，女佣也看不到未来安排，没法提前备料采购。

## 目标

- 雇主可以提前把本周（周一~周日）任意一天的三餐排好，操作方式与现在"安排到今日"完全一致，选好即时生效。
- 雇主 / 女佣双方首页都能看到"本周"视图，快速切换查看某一天的菜单。
- 女佣端提前看到未来安排，方便提前采购备料。

## 非目标

- 不支持切换到上一周/下一周，只展示当前自然周（周一~周日）。
- 不引入"草稿 → 一键发布"机制，安排即生效（与现状行为一致）。
- 不改变 `MealOrder` 的状态流转（to_receive/received/… 等生命周期不变）。

## 设计

### 1. 日期范围

一周固定为「周一 ~ 周日」的自然周，包含当前日期所在的这一周。不提供"上一周/下一周"导航。若当天是周日，视图仍显示本周一~本周日（不会因为周日快过去了就跳到下周）。

### 2. UI：日期条 + 滑动切换

`EmployerHome` 与 `MaidToday` 的"今日菜单"卡片改为如下结构：

```
一   二   三   四●  五   六   日
14   15   16  [17]  18   19   20
──────────────────────────
🍳 早餐：三明治
🍚 午餐：番茄炒蛋饭
🍲 晚餐：未安排
```

- 顶部一排 7 个日期胶囊（周几 + 日期数字），默认高亮当前选中日（初次进入 = 今天）。
- 胶囊上若当天已有安排（至少一餐有菜），加一个小圆点标记；空的不加标记，方便一眼看出本周哪几天还没排。
- 点击胶囊 = 直接跳到那一天；在下方卡片内容区左右滑动 = 切换前一天/后一天，胶囊高亮联动更新。
- 滑动/切换只影响本地展示状态（当前查看哪一天），不涉及写操作。
- 员工端和女佣端复用同一套日期条组件；女佣端保持只读（不显示"删除菜品"等雇主操作）。

### 3. 安排入口

沿用现状：仍从 `RecipeDetail.jsx` 的"安排到菜单"入口发起。原来的餐次选择（早/午/晚）旁边，日期选择从固定"今天"改成本周 7 天的选择器（复用日期条 UI 或简单的 7 个可点选项）。选好日期 + 餐次后立即调用 `recipeToMeal` 写入，不做草稿暂存,女佣端首页刷新即可看到。

### 4. 后端改动

- `GET /meals`（或对应 bootstrap 聚合接口）需要按 `meal_date` 过滤本周范围（周一~周日），而不是只取当天，返回本周全部 7 天的数据供前端按天分组展示。
- `recipeToMeal` 的请求体新增/复用 `meal_date` 字段（原来隐式为当天，现在由前端传入本周内选中的具体日期），后端做一次范围校验（必须落在本周 7 天内）。
- 已有的删除菜品（`deleteMeal`）、状态流转接口不变。

## 测试关注点

- 周一/周日边界：确认"本周"计算在一周的头尾两天都正确（不多算/漏算）。
- 时区：与现有"今日"判断一致，使用服务器已统一的本地时区（新加坡）计算周边界，避免跨时区导致周一算错。
- 女佣端只读展示未来某天的菜单时，不应误触发任何写操作。
