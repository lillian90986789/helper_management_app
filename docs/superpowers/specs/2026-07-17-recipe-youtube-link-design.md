# 菜谱 YouTube 链接 — Design

## 背景

`RecipeStep` 表已有按步骤挂视频的 `video_url` 字段，但 `Recipe`（菜谱整体）本身没有视频链接字段。用户希望菜谱可以关联一个 YouTube 教程链接，点击跳转到 YouTube 观看。

## 目标

菜谱整体可以配置一个 YouTube（或任意视频网站）链接，在菜谱详情页展示一个入口按钮，点击跳转到该链接。

## 非目标

- 不做站内嵌入播放（不用 iframe 嵌 YouTube 播放器），只做外部跳转。
- 不按步骤（`RecipeStep.video_url`）扩展，只在菜谱整体层面加一个链接。
- 不校验链接格式是否为合法的 youtube.com/youtu.be 链接，任意 URL 都接受，为未来支持其他视频站点留空间。

## 设计

### 1. 数据模型

`Recipe` 表新增一列：

```sql
ALTER TABLE Recipe ADD COLUMN video_url TEXT;
```

不做非空/格式校验。

### 2. 创建/编辑

`RecipeNew.jsx`（新建/编辑菜谱表单）新增一个可选输入框"视频教程链接"，直接存文本，不做 URL 格式校验，允许留空。

### 3. 详情页展示

`RecipeDetail.jsx` 封面图附近新增一个按钮，例如：

```
▶️ 观看 YouTube 教程
```

仅当 `recipe.video_url` 非空时显示该按钮；点击用 `target="_blank"`（或 App 内置浏览器同等打开方式）跳转到该链接，不做内容预取或解析。

### 4. 后端改动

- `Recipe` 相关的创建/更新接口的请求体新增可选字段 `video_url`，直接透传写入/更新，无需额外校验逻辑。
- 查询菜谱详情/列表的接口返回值里带上该字段即可（现有 `SELECT *` 类查询大概率已自动包含新列，需要确认没有显式字段白名单遗漏它）。

## 测试关注点

- 未填链接时按钮不显示，不出现空链接的死按钮。
- 已有旧菜谱（无该字段历史数据，`video_url` 为 NULL）详情页正常展示，不报错。
